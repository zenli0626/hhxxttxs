## ✅ 今日可行动 / Do this today

- 今晚按『一次严肃攻击现在只要 25 美元』重估一遍你手上任何对外的东西。Searchlight 的研究员把 OpenAI 公开的那条解数学猜想的提示词改了改，指向 WordPress、开四个 agent 跑六个钟头，就在默认配置里挖出一条从未授权直达 RCE 的链；这种掮客出价约五十万美元的漏洞，他一共花了约 25 美元，还被另外两拨人独立复现了。它作废的是『没人会费劲盯上我这点规模』这条老假设：盯上你的边际成本已经趋近于零。做三件具体的。一，你要真跑 WordPress，今晚就去 wp2shell.com 自查一遍、该打补丁打补丁。二，把你所有对外接口和上传点列出来，假设它们本周都会被一个多 agent 扫一遍，先自己按同样思路（多路并行探索、每个疑点反复自检、别在第一波无果就停）审一遍。三，把『我们太小众不值得被攻击』这句话从威胁模型里删掉。 *([Hacker News · WordPress 预认证 RCE：$500k 的漏洞，$25 挖到](https://news.ycombinator.com/item?id=48975665))*
- 你要在做带 AI 的产品，这周逼自己画一版不是聊天框的界面。Kimi Work 这类『知识工作 agent』几乎清一色摆个大聊天框，HN 上被一句话点破：大家默认上聊天框，不是因为它最好，是因为它最安全、最符合预期、最不用重新想。可这恰好把最难也最值钱的活儿躲过去了。做一件事：挑你产品里一个高频场景，别问『用户会在聊天框里打什么』，改问『这个 agent 要把活干好，需要我提前把哪些上下文喂到它面前，又该把它的产出摆在哪、让人一眼能核』。把这一屏画出来，先是纸上草图也行。今天满版都在说值钱的是模型外面那层安排，界面就是你能亲手搭的那层，别默认交给一个输入框。 *([Hacker News · Kimi Work](https://news.ycombinator.com/item?id=48981703))*
- 给自己立一条规矩：以后看到任何『X 有几成已经是 AI』的数字，先问它的误报地板是多少，再决定信不信。unslop 那篇量 arXiv 的做法值得抄：它没直接甩出『三分之一新论文像是 AI 写的』，而是先拿 2021–2022 年、ChatGPT 之前的老论文当铁定的人写样本，把阈值卡到只有 0.4% 的老论文会被误标，再在这条地板之上去报别的年份和学科（计算机 65%、数学 0.7%）。没有这条地板，任何『几成是 AI』的标题都可能只是探测器自己的误报率在冒充趋势。这条今晚就能用：你手上任何一个『AI 占比 / AI 命中率 / AI 贡献度』的内部指标，补一次『拿一批确定不含 AI 的历史样本，看这套算法会误标多少』；误报地板报不出来，这个数就先别往上报。 *([Hacker News · 我们怎么量 arXiv 上的 AI 写作，又在哪里失准](https://news.ycombinator.com/item?id=48981206))*

## 🧭 本报社评 / Editorial read

把今天几条并排看。Cursor 拿同一个任务（照 835 页文档从零用 Rust 写 SQLite），换不同模型搭配跑，质量都差不多，账单却从 1339 美元到 10565 美元，差了近八倍；同一批活，光是把规划交给强模型、执行交给便宜模型，worker 那队就从 9373 美元掉到 411 美元。Searchlight 那位研究员更直接：把 OpenAI 公开的解数学猜想提示词改一改、指向 WordPress、开四个 agent 跑六小时，就在默认配置里挖出一条从未授权打到 RCE 的链，掮客开价约五十万美元的东西，他花了约 25 美元。Kimi Work 这类知识工作 agent 一水儿摆个聊天框，HN 上有人点破：默认上聊天框只因为那是最安全、最不用重想的选择，真正难的是重想界面。unslop 量 arXiv 那篇最见功夫的也不是探测器，是它先把 ChatGPT 之前的老论文卡在 0.4% 的误报地板上，再报别的数。市面把这些分开读：有人赚大钱、有人几十美元干成大事、人人上聊天框、三分之一论文是 AI 写的。本报的另一种读法是：模型本身已经基本是一份可换的原料，今天每一条都在指同一件事，值钱的和稀缺的都挪到了模型外面那层安排上。Cursor 自己把话说白了，swarm 更像个编译器，真正稀缺的是『把意图说清楚的那份说明书』；WordPress 那条稀缺的是那条提示词，不是模型；Kimi Work 稀缺的是敢不摆聊天框的那版界面；unslop 稀缺的是那条老实的误报地板。所以昨天说护城河搬出了模型、搬进了『价值归谁』和『有没有人诚实计量』，今天再往前挪一格：它搬进了『你怎么把模型摆起来用』这层安排，规格、编排、界面、计量，四样都是安排，都得自己搭，没人替你搭（这层安排怎么搭得省，详见头版与今日所学）。Altman 一封被翻出来的旧邮件正好从反面印证这层安排的分量：他当年想抢在别人前面放一个能在消费级硬件上跑的开源模型，图的不是开放，是『让别人更难放出同等模型、更难拿到融资』；开源在他那儿从来是一步棋，不是一份礼物。今天中国那批开源权重被读成在赢，赢的也不是权重本身，是谁把这步棋走进了自己的生态安排里。对独立建设者，能拿的东西还是在这层安排上，不在最强权重：强模型只留给真正要判断的少数几步、其余交给便宜模型（详见今日所学）；威胁模型按『一次攻击只要 25 美元』重估（详见一版行动）；界面别默认成聊天框（详见二版行动）；信任任何『几成是 AI』的数字前先要它的误报地板（详见三版行动）。这些都不花钱，花的是肯自己搭那层安排。

*Line up today's items. Cursor ran the same task (build SQLite in Rust from 835 pages of docs) across different model mixes: quality was similar, but the bill ran from $1,339 to $10,565, nearly eightfold, and for the same work, just moving planning to a strong model and execution to a cheap one dropped the worker team from $9,373 to $411. The Searchlight researcher is even blunter: adapt OpenAI's published math-conjecture prompt, point four agents at WordPress for six hours, and out of the default config comes a pre-auth-to-RCE chain — a bug brokers pay ~$500k for, found for ~$25. “Knowledge-work agents” like Kimi Work all ship a chat box, and HN nailed why: the chat box is the default only because it's the safest, least-rethinking option, while the hard part is rethinking the interface. And the sharpest thing in unslop's arXiv study isn't the detector, it's that it pinned pre-ChatGPT papers to a 0.4% false-positive floor before reporting anything else. The market reads these separately: someone makes big money, someone does a big thing for pocket change, everyone ships a chat box, a third of papers are AI. This paper's other reading: the model itself is now basically an interchangeable input, and every item today points at the same thing — the value and the scarcity have moved out to the arrangement around the model. Cursor says it plainly: the swarm is more like a compiler, and what's scarce is “the right description of intent”; for the WordPress bug the scarce thing was the prompt, not the model; for Kimi Work it's the interface that dares not to be a chat box; for unslop it's that honest false-positive floor. So yesterday's “the moat left the model, into who-gets-the-value and who-measures-honestly” advances one notch today: it moved into how you arrange the model in use — spec, orchestration, interface, measurement, all four are arrangement, and you have to build each yourself (how to build that arrangement cheaply, see the front page and today's learnings). A resurfaced old Altman email confirms the weight of that arrangement from the other side: he once wanted to beat rivals to releasing a consumer-hardware open model not out of openness but to “make it harder for others to release similarly-powerful models and harder for new efforts to get funded” — open source, to him, was always a move, not a gift. Today's Chinese open weights are read as winning, but what's winning isn't the weights, it's who worked that move into their own ecosystem's arrangement. For solo builders, what you can grab is still in this arrangement layer, not in the strongest weights: reserve the strong model for the few steps that need judgment and hand the rest to cheap ones (see today's learnings); re-price your threat model at “an attack costs $25” (see Action 1); don't default the interface to a chat box (see Action 2); demand a false-positive floor before trusting any “N% is AI” number (see Action 3). None of this costs money; it costs being willing to build that arrangement yourself.*

## 🆕 今日所学 / What I learned

- 把这条升级阶梯上最近的几条经验并排看，它们全长在同一根轴上：从二元评测、按努力档升级、弃权门、多供应商 fail over，到昨天的『成本诊断先于升级』，讲的都是子任务失败后该怎么加力、以及加力前先量一下成本。今天 Cursor 那篇《Agent swarms and the new model economics》给了这根轴一个此前一直没说死的结构性默认：同一个任务、同样的质量，只看模型搭配，账单能从 1339 美元差到 10565 美元；把规划交给强模型、执行交给便宜模型之后，worker 那队从 9373 掉到 411 美元。它把话点破了——一个大任务里真正需要前沿智能的只有那几步（最初的分解、设计决策、少数关键取舍），一旦强模型把含糊收敛成明确指令，后面的执行换便宜模型『照做就行』。
- 也就是说，此前那条成本诊断是『失败后先量再决定升不升』，但更省的其实在更前面：一开始就该按角色分档，而不是等失败了再补救。由此可以合成一条『按角色分档，而非按调用升档』：默认就把前沿模型只留给真正含糊、需要判断的少数几步（分解、定契约/接口、定关键取舍），把所有执行/worker 的 token 默认压到能过评测的最便宜模型上，而不是让整条链跑在一个强模型上、再靠后面的诊断和逐级升档去救。它跟前面几条不同——那些是『失败之后怎么办』，这条是『开工就把强算力花在刀刃上』的默认分配，Cursor 那组 411 对 9373 的数字就是这条默认能省下的量级。
