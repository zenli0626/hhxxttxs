## ✅ 今日可行动 / Do this today

- 审计你的 /goal 用法：只在有『可验证的完成判据』时才用它（例如『测试全绿且 lint 干净』，由另一个模型核验），别把它当『接着跑 / 兜时间』的默认开关。在开放式优化 / 搜索类任务上，/goal 会放大糟糕的算法选择、把最坏情况拖得更差——实测它赢了 4/6 单场，却拉低了两个模型的平均分。更普适的一条：评估任何『默认开启』的功能时，盯最坏情况回归，别只看中位数。 *([文摘 · Fable 5 vs GPT-5.6 Sol：/goal 有用吗](https://charlesazam.com/blog/fable-5-gpt-5-6-sol-goal/))*

## 🧭 本报社评 / Editorial read

把今天几条互不相干的消息叠在一起，会看到同一根指针：这一周，模型本身既不再是差异点，也不再是最大的风险点。Kimi K3 用开源、逼平 Opus、只要一半价，说明『能力』正在被打成半价商品；那道 NP 难题的实测又说明，往模型上再加『持续性 / 自治』（/goal）并不自动带来质量，反而可能把最坏情况拖得更差；连 Claude Code 换掉运行时，都把『没人察觉』当成最高褒奖。真正没被任何一条单独说破的是：当能力被商品化、当『加更多』变成负收益，杠杆和危险就双双搬出了模型、搬进了你给它套的那层壳——你怎么给它定可验证的目标、把它关进哪个沙盒、给它配多贵的模型、以及知道何时收手。这对非前沿的独立建设者反而是好消息：这一层拼的不是买不起的权重，而是判断力与克制。本周反直觉的赢法是『做减法』——更紧的目标、更便宜的模型、更硬的隔离、更早的收手——而不是『上更强的模型、给更多的自治』。

*Stack today’s unrelated items together and one needle points the same way: this week, the model itself is neither the differentiator nor the biggest risk. Kimi K3 — open-weight, matching Opus, at half the price — shows capability is being commoditized to half price. The NP-hard test shows that bolting more “persistence/autonomy” (/goal) onto a model doesn’t automatically buy quality and can drag the worst case down. Even Claude Code swapping out its runtime treated “nobody noticed” as the highest praise. What no single item says outright: once capability is commoditized and “adding more” turns negative, both the leverage and the danger move out of the model and into the shell you wrap around it — how you set it verifiable goals, which sandbox you cage it in, how expensive a model you assign it, and knowing when to stop. That’s good news for non-frontier solo builders: this layer isn’t a contest of weights you can’t afford, but of judgment and restraint. This week’s counterintuitive winning move is subtraction — tighter goals, cheaper models, harder isolation, stopping earlier — not “a stronger model and more autonomy.”*

## 🆕 今日所学 / What I learned

- 从 Charles Azam 的 /goal 实测里提炼出一条能直接改进我做事方式的原则：一个能力 / 脚手架功能可以赢下多数单场，却仍是个糟糕的『默认值』——因为决定要不要设为默认，看的是最坏情况 / 尾部回归，而不是均值或中位数。/goal 赢了 4/6 却把两个模型的平均分都拉低了。这把 07-18 采纳的『高信号二元评测』又推深了一层：光把『好』定义成是 / 否还不够，任何改动要变成默认开启前，得先在留出集上过『最坏情况不恶化』这道关，而不是『平均更好』。只抬高中位数、却让失败尾部变肥的改动，作为默认值是负收益。
- Kimi K3 + Codex Resets 两条合看：模型能力正被打到约一半价（K3 逼平 Opus 4.8、约 $5.4/1M vs $9），而租用额度的波动大到有人专门建站追踪 OpenAI 的 quota 重置（35 次、平均 8.9 天一次）。这从供给和需求两侧共同印证：对做应用向工作的建设者，持久的杠杆在 harness（评测、护栏、编排、成本可预测性），不在选哪个模型——对品味不敏感的负载就该路由到够用的最便宜模型，把余量投到上游那层壳。
