## ✅ 今日可行动 / Do this today

- 别再让 AI 一键概括长文——挑一篇你一直想认真读的（比如 PG 的《How to do great work》），用沉浸式方式慢读一遍。深读本身就是最好的 eval，也是判断层能力的日常训练 *([小红书 · 给 Paul Graham 做沉浸式阅读站](https://www.rednote.com/explore/6a42f61c0000000008033549))*
- 如果你在跑任何 agent 循环/自动化流程，今晚给它补一层真正的验证——强类型或一套测试。三场独立信源(DeepMind Benoit、YC、Loops 辩论)都指向同一句话：机械的、spec 明确、有测试覆盖的切片可以交给 loop 无人值守，但『建得对不对』要人判断；没有验证的 loop 就是『马戏团』 *([YouTube · The Great Loops Debate](https://www.youtube.com/watch?v=c35YoMdnI78))*
- 给你的 agent 建一个最小『公司大脑』：每次会话结束把学到的东西写下来（带来源），别让明天的 agent 变成金鱼。适用于任何长期跟 Claude 协作的人——Garry Tan 的判断是『模型质量是租来的，你自己攒的 context/记忆层才是你拥有的』，决定 agent 是天才还是金鱼的，是谁来决定它开着哪三本书 *([YouTube · Garry Tan / Y Combinator](https://www.youtube.com/watch?v=eBUyTS7SzV4))*

## 🧭 本报社评 / Editorial read

把今天所有信源叠在一起，只有一句是它们共同的底层：执行层在贬值，判断层在升值——写代码、写详细提示词这类「动手」正在被 agent 拉平，而品味、eval、目标设计这类「定标准」成了新的稀缺。但更值得记下的是一道中外温差：英文圈在认真讨论怎么把 agent 编成千人团队、怎么给模型少约束多目标（生产力上限的话题）；中文圈的情绪却明显倒向 OpenAI/Codex，连 Anthropic 那支正视风险的广告都被当成「AI 末日预告」群嘲。一边在谈能力天花板，一边在谈消费情绪——这个错位本身，就是这一期最该带走的信号。

*Stack every source today and one thing sits underneath all of them: execution is being commoditized, judgment is appreciating. Writing code and writing detailed prompts — the “doing” — is being flattened by agents, while taste, evals, and defining the goal — the “setting the bar” — became the scarce part. But the sharper takeaway is a temperature gap between East and West: the English-speaking side is seriously debating how to staff 1,000-agent teams and how to give models fewer constraints and clearer goals (a ceiling-of-capability conversation); the Chinese side leans emotionally toward OpenAI/Codex, and even Anthropic’s risk-facing ad got mocked as an “AI apocalypse trailer.” One side is talking capability, the other is talking sentiment — that mismatch is itself the signal worth carrying out of this issue.*

## 🆕 今日所学 / What I learned

- Garry Tan(YC)的『新商业物理学』：同一个人同一个 Claude，2x 的人和 100x 的人差距不在模型权重，而在『怎么编排工作』——他把 agent 工程整套映射成一家公司：skill 文件=员工、resolver 表=组织架构、filing rules=流程、trigger eval=绩效考核；context/记忆层=『公司大脑』(图书馆+图书管理员，带来源标注和矛盾检查)。这套心法和常见的『manager + worker agent』编排模式完全同构，可以直接拿来系统化
- Benoit Schillings(DeepMind 研究 VP)和 Loops 辩论交叉印证的一条硬结论：强类型是一种验证形式。用 Python/Ruby 跑 agent loop 会变『马戏团』，Rust/Haskell 这类强类型语言可维护性显著更好；评测也要从『代码能不能跑』升级到开放式问题(Benoit 的无损压缩器例子)。给本报的启发：以后为任何自动化流程设计验收时，优先靠类型/测试这种结构化验证，而不是靠肉眼读输出
