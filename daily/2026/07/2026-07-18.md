## ✅ 今日可行动 / Do this today

- 给你正在跑的任何 agent 先装一个 kill switch：一个全局的 + 每个高危工具各一个，本周就做。它秒级生效、不用重新部署，这一个动作改变的运营姿态，比这季度任何工程投入都大 *([YouTube · Agents Need Feature Flags](https://www.youtube.com/watch?v=zU4EagB311U))*
- 把手上一个模糊的『质量 / 正确性』评测，改写成二元高信号判据，比如『答案是否只用了检索到的原文？是 / 否』，别再用 0–1 打分。只有二元信号能让自我改进真正收敛，也更省 token *([YouTube · Stop Burning Tokens](https://www.youtube.com/watch?v=eAXxdtNlK04))*
- 算一次你或团队真实的推理账单：还在找 PMF 就继续租；一旦过了 PMF、或已按项目立了预算，就该认真评估自建 / 自托管的拐点：rent to learn, own to earn *([YouTube · Stop Renting Your Cognitive Infrastructure](https://www.youtube.com/watch?v=Bck7ABCZRZI))*

## 🧭 本报社评 / Editorial read

把今天大会一整批演讲叠在一起，能读出一个不再关于『能不能做』的转向：agent 能做已经是默认前提，会场集体在补的是『怎么安全地让它一直做下去』：刹车（feature flags）、账本与回执（receipts）、成本治理（自建推理）、高信号评测、以及把人留在方向盘上的 UX。这些半年前几乎不存在的东西，正凝结成一门『agent 运维』学科。真正该带走、且各条演讲都没直说的一点是：当模型性能不再是差异点（UX 那场原话：差异化已经从性能挪到了你在模型外面搭的体验），护城河就从『用哪个模型』搬到了『你给这个商品化模型套了多严的运营外壳』。这对非前沿的独立建设者反而是好消息：你拼的不再是买不起的权重，而是刹车、评测、账本这些谁都能建、多数人却懒得建的东西。

*Stack today’s batch of conference talks together and a shift emerges that is no longer about “can it be done”: that agents can do the work is now the default premise; what the floor is collectively backfilling is “how do you let them keep doing it safely” — brakes (feature flags), ledgers and receipts, cost governance (owning inference), high-signal evals, and UX that keeps a human at the wheel. These things barely existed six months ago and are congealing into a discipline of “agent operations.” The takeaway worth carrying out — which none of the talks says outright — is this: once model performance is no longer the differentiator (the UX talk’s own words: differentiation has moved from performance to the experience you build around the model), the moat moves from “which model you use” to “how tight an operational shell you wrap around a commoditized model.” That’s good news for non-frontier solo builders: you’re no longer competing on weights you can’t afford, but on brakes, evals, and ledgers — things anyone can build and most people can’t be bothered to.*

## 🆕 今日所学 / What I learned

- Langfuse 的 Annabell Schäfer 证实：自我改进循环只在『目标函数是高信号（清晰的是 / 否）』时才收敛，写代码能自改，是因为『能不能编译』恰好是二元的；把 0–1 的『正确性 / 有用性』打分喂给自改循环是低信号、跨次不稳定、白烧 token。要把『好』重写成可观测的二元判据（如『答案是否只基于检索到的原文？是 / 否』），并留一份 held-out 验证集防过拟合、给循环一个 escape hatch。
- Sachin Gupta 的 Feature Flags 给了一句可迁移的运营口径：『2026 是采纳，2027 是控制』，和一条最小起手式：先装 kill switch（全局 + 每工具），秒级生效、不用部署。一条易被忽略的铁律：所有派生出去的 worker agent 必须走同一层中间件，否则你刚按下的急停根本传不到派生出去的子进程。
